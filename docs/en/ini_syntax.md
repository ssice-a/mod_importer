# INI Syntax

English | [中文](../ini_syntax.md)

This document describes the NTMI fast-path INI generated by the add-on.

The old `ShaderOverride + checktextureoverride = ib`, ShaderRegex, BoneStore, PoseSlot, and per-character HLSL copy workflow is no longer supported.

## Runtime Flow

```text
Original game skin CS
-> Collector records original bone atlas pieces
-> TextureOverride hit asks the Collector to build for the current vb0 instance
-> CommandList_SkinParts_* calls NTMI Core skinning
-> Fast TextureOverride replaces IB/VB/VS/PS resources and draws the custom mesh
```

## TextureOverride

A draw region is matched by original IB hash, `match_first_index`, and `match_index_count`.

```ini
[TextureOverride_IB_0456d530_105840_70866]
hash = 0456d530
match_first_index = 70866
match_index_count = 105840
handling = skip
collector = CollectorSkinPart_0456d530, vb0
ib = ResourcePart_0456d530_105840_70866_part00_IB
match = vb, dynamic, ResourcePart_0456d530_105840_70866_part00_RuntimeSkinnedPositionVB
match = vs, dynamic_prev, ResourcePart_0456d530_105840_70866_part00_RuntimePrevSkinnedPosition
match = vs, 9393e3bb, ResourcePart_0456d530_105840_70866_part00_Texcoord
match = vs, 9130d548, ResourcePart_0456d530_105840_70866_part00_Position
match = vs, dynamic, ResourcePart_0456d530_105840_70866_part00_RuntimeSkinnedNormal
match = vs, d77b480e, ResourcePart_0456d530_105840_70866_part00_OutlineParam
ps-t7 = ResourceTexture_0456d530_105840_70866_T7
drawindexed = 77442,0,0
```

Rules:

- `hash` is the currently bound original IB resource hash.
- `match_first_index` and `match_index_count` must be exact.
- `collector = CollectorName, vb0` triggers the Collector for the current original `vb0` instance.
- `handling = skip` skips the original draw.
- `ib = ...` binds the exported custom IB.
- `drawindexed = index_count,start_index,base_vertex` draws a slice from the custom IB.

## Match Syntax

```ini
match = vb, dynamic, ResourcePart_*_RuntimeSkinnedPositionVB
match = vs, dynamic_prev, ResourcePart_*_RuntimePrevSkinnedPosition
match = vs, <resource_hash>, ResourcePart_*_Texcoord
match = vs, <resource_hash>, ResourcePart_*_Position
match = vs, dynamic, ResourcePart_*_RuntimeSkinnedNormal
match = ps, <resource_hash>, ResourceTexture_*
```

- `dynamic` uses the original runtime resource pointer recorded by the current Collector group.
- `dynamic_prev` reads the previous-frame slot for the current position resource.
- Static hash matches replace resources such as texcoord, position, outline parameters, or textures.
- PS textures can be bound explicitly with `ps-tN = Resource...` or via `match = ps, hash, Resource...` when appropriate.

## Collector

The Collector gathers the original game bone atlas and maps original runtime outputs to exported dynamic resources.

```ini
[CollectorSkinPart_0456d530]
group = cs-u1
match_cs_t0_hash = f5def9cf
match_cs_u0_hash = c0db46f4
match_cs_u1_hash = fa3995da
collect = write, cs-t0, cs-cb0[1]
build = Resource\NTMIv1\RuntimeGlobalT0
map = cs-u1:ResourcePart_0456d530_105840_70866_part00_RuntimeSkinnedPosition, cs-u1:ResourcePart_0456d530_105840_70866_part00_RuntimeSkinnedPositionVB, cs-u0:ResourcePart_0456d530_105840_70866_part00_RuntimeSkinnedNormal
run = CommandList_SkinParts_0456d530
```

- `group = cs-u1` usually groups by the original skinned position output resource.
- `match_cs_*_hash` values are resource guards, not shader hashes.
- `collect = write, cs-t0, cs-cb0[1]` records the current bone atlas input by a stable CB key.
- `build = Resource\NTMIv1\RuntimeGlobalT0` builds the global bone matrix pool for the current group.
- `map = ...` maps original output resources to exported dynamic resources.
- `run = CommandList_SkinParts_*` runs custom skinning after build.

The analyzer should trace the actual `vb0` used by the target g-buffer draw back to its producer CS output pool. Do not infer the bone chain only from VS/PS shader hashes.

## Skin CommandList ABI

- `cs-t64`: Collector-built global bone matrix pool.
- `cs-t65`: current part palette.
- `cs-t66`: packed blend index/weight.
- `cs-t67`: static normal/frame input.
- `cs-t68`: static position input.
- `cs-t69`: optional shapekey static data.
- `cs-t70`: optional shapekey runtime weights.
- `cs-u6`: runtime skinned normal output.
- `cs-u7`: runtime skinned position output.

`$\NTMIv1\vertex_count` is the exported part vertex count, not the index count.

## Palette Rules

Every exported part has its own local palette:

```text
Blend local bone index -> Palette local slot -> RuntimeGlobalT0 global bone index
```

- Blender vertex groups should use global bone ids before export.
- Export remaps each part to local bone indices starting at 0.
- The palette stores local-to-global mapping.
- One exported part cannot exceed 256 local bones because blend indices are 8-bit.
- IB can be exported as `R16_UINT` or `R32_UINT`; vertex count is no longer limited to 65535.

## Shapekey

Runtime shapekey export is optional.

Static position and normal buffers always represent the visible Blender result at export time. Runtime shapekey data stores deltas relative to that exported visible state, so the initial game result does not apply the same shapekey twice.
